# Статус рефакторинга NWN Modules Translator

Этот документ отслеживает прогресс устранения технического долга, архитектурных проблем и проблем производительности в кодовой базе. Одобрен к выполнению порциями по запросу разработчика.

## ✅ Выполненные задачи (Завершено)

### 🔴 Критические ошибки логики
- **[Д-1] Мутация `node.text` при sanitize**: Исправлено через использование `text_overrides` в формате диалогов (нарушение PoLS устранено, оригинальные структуры GFF остаются целыми).

### ⚡ Производительность и Асинхронность
- **[Perf] Устранение зависаний пайплайна (Rate Limit & Concurrency)**:
  - Разрешена проблема невидимого Rate Limit 429 от API OpenRouter путем интеграции `@retry` с явным логированием `before_sleep` из пакета `tenacity` и отключения тихих ретраев `openai` SDK (`max_retries=0`).
  - Исключен баг `APIConnectionError` благодаря созданию thread-local пула соединений (`AsyncOpenAI` инстанцируется с привязкой конкретно к `event_loop` текущего потока).
  - Внедрен `ThreadPoolExecutor` для конкурентной многопоточной обработки файлов, что кратно повысило утилизацию сети на мелких файлах. Файлы обрабатываются параллельно.
  - Внедрены `threading.Lock()` для потокобезопасного мержа статистики и кэша из разных потоков.
- **[11] Замена ручного event loop**: Решено. В `TranslationManager` небезопасный глобальный `self._loop` был заменен на изолированные циклы через утилиту `run_async()`.

### 🗑 Мёртвый код (Удалено)
- **[1] Функция `translate_file()`**: Заменена вызовами `ModuleTranslator`.
- **[2] Методы `_create_local_string`, `_update_local_string` в `BaseInjector`**: Не использовались.
- **[3] Неиспользуемый `config` в Base классах Extractor/Injector**
- **[4] Скрипт `scripts/dump_ifo.py`**
- **[6] `extract_tokens()` обёртка в `token_handler`**
- **[7] Неиспользуемый атрибут `self.token_handler` в `ContextualTranslationManager`**
- **[18] `translate_batch` из `BaseAIProvider` и абстракций**: Провайдеры переведены на асинхронный поштучный/concurrent перевод.

### ⚙ Технический долг
- **[14] Версия пакета в одном месте**: Теперь `__version__` загружается динамически из `pyproject.toml` в `__init__.py` и `cli.py` (через `importlib.metadata`).
- **[15] Удаление Placeholder URL**: Заглушечные URL удалены из `pyproject.toml`.
- **[16] Отсутствие `__init__.py` в `context/`**: Файл добавлен для корректного форматирования пакета модулей.

### 🏗 Инкапсуляция и PoLS
- **[Д-2] Инкапсуляция `_translation_cache`**: Переименован `_translation_cache` → `translation_cache` (публичный атрибут, т.к. используется из оркестратора).
- **[17] Переименование `_cleanup_and_return`**: Переименован → `_resolve_output_path`, обновлён docstring.

### 🏛 Архитектура и SRP
- **[9] DRY для `_lang_suffix`**: Единственный источник `lang_suffix()` в `config.py`; `cli.py` и `web/task_manager.py` импортируют его.
- **[10] Singleton-маппинг слоёв парсинга**: Заменено на статические `_EXTRACTOR_MAP` / `_INJECTOR_MAP`, строящиеся при импорте. Экземпляры переиспользуются.
- **[12] Вынос `_extract_text_from_local_string`**: Вынесен в модульную функцию `extract_local_string()`; `DummyExtractor` хак в `WorldScanner` удалён.
- **[8] Упрощение оркестратора**: Delta-tracking статистики консолидирован в `_sync_manager_stats()` с корректной блокировкой через `_stats_lock`.

### 🖥 CLI и точки входа
- **[19] Унификация языка CLI help**: Описания команды `web` переведены на английский.
- **[13] Удаление дублирующих точек входа**: Удалён `scripts/run_web_ui.py`. `run-web-ui.bat` упрощён до вызова `nwn-translate web --reload` + Vite напрямую.
